# Project Memory: Agentic Research RAG

## 🚀 Tech Stack
- **Backend:** Python, FastAPI, Qdrant (Local Path Mode), OpenRouter
- **Frontend:** Angular 21, Tailwind CSS v4, PrimeNG
- **Storage:** Local `qdrant_db` (Vectors) + IndexedDB `AgenticRagDB` (Chat History)

## ✨ Current Capabilities

### RAG & Document Processing
- **Robust Extraction:** High-fidelity text and table parsing via `PyMuPDF` + `pdfplumber`.
- **OCR Fallback:** `EasyOCR` integration for scanned/image-heavy PDF pages (weights pre-loaded at startup).
- **Contextual Chunking:** Section headings are merged into chunk text (`heading\n\nbody`) to preserve metadata without complex retrieval patterns.
- **Deduplicated Citations:** Backend filters prevent duplicate source tags for multiple chunks from the same file.

### Chat & Persistence
- **Local History:** Full conversation persistence via `IndexedDB`. Messages and retrieved context remain available across refreshes.
- **Session Management:** Dedicated sidebar for creating, switching, and deleting chat sessions. Titles are auto-generated from the first query.
- **Deep System Reset:** Synchronized "Clear Index" action that wipes both Qdrant vectors and local IndexedDB history.

### UI/UX (Premium Black + Orange)
- **3-Pane Layout:** Sidebar (History/KB), Center (Chat), Right (Evidence/Citations).
- **"Google Search" Bar:** Floating pill-shaped input with integrated model selection and action chips.
- **Granular Feedback:** UI tracking for "Searching", "Retrieving", and "Synthesizing" stages.
- **Shortcuts:** `Ctrl+B` (Toggle History/KB), `Ctrl+Alt+B` (Toggle Evidence).

## 🗺️ Key Code Paths

### Backend (`/backend/app`)
- `api/chat.py` -> Streaming SSE endpoint + retrieval orchestration.
- `api/documents.py` -> Upload & index management.
- `services/document_processor.py` -> PDF parsing, EasyOCR, and logical chunking.
- `services/injest.py` -> Qdrant collection management and embedding loops.

### Frontend (`/frontend/src/app`)
- `services/chat.service.ts` -> Streaming & state bridge.
- `services/chat-history.service.ts` -> IndexedDB persistence layer.
- `components/app-shell/` -> Layout and sidebar orchestration.
- `components/chat-panel/` -> Main conversation workspace.
