## Current RAG Pipeline Architecture
**1. Document Processing (`backend/app/services/document_processor.py`):**
- Documents are parsed, split into logical chunks.
- A **Parent-Document Retrieval (PDR)** structure is utilized. Smaller chunks are created for precise vector search, but they maintain a `parent_id` linking them back to the larger surrounding context.

**2. Vector Searching and Context Assembly (`backend/app/api/chat.py` & `faiss_service.py`):**
- User queries are embedded via `get_embedding`.
- A FAISS exact search retrieves the top `k=20` chunks.
- Instead of feeding just the small chunks to the LLM, the pipeline resolves the `parent_id` for each chunk to fetch the **full parent text block**.
- The parent blocks are deduplicated (preventing redundant tokens if multiple chunks from the same parent are found) and stitched together to form a rich context `faiss_manager.build_context`.

**3. Streaming response (`backend/app/api/chat.py`):**
- First, the system yields the deduplicated parent sources via Server-Sent Events (SSE) using the `type: sources` JSON format. (Including source name, page numbers, text snippets, and similarity scores).
- Next, the `generate_answer_stream` connects to OpenRouter, feeding it the user's query and the assembled parent-context. The response tokens are streamed back to the Angular UI as they are generated.

**Backend (FastAPI Engine):**
- `backend/app/api/chat.py` -> API Endpoints for handling Chat requests and streaming.
- `backend/app/api/documents.py` -> API Endpoints for uploading and inspecting documents.
- `backend/app/services/llm.py` -> The OpenRouter LLM connection and request formatting.
- `backend/app/services/faiss_service.py` -> Creating, saving, and querying FAISS embeddings.
- `backend/app/services/document_processor.py` -> PDF/Document parsing and chunking. 

**Frontend (Angular UI):**
- `frontend/src/app/components/chat-panel/` -> The main conversational hub.
- `frontend/src/app/components/left-panel/` -> The document management / summary view.
- `frontend/src/app/components/right-panel/` -> The citation/evidence review area.
- `frontend/src/app/components/message-list/` & `chat-input/` -> Micro-components for message flow.
- `frontend/src/app/services/chat.service.ts` -> Core service bridging Angular with the FastAPI endpoints.
- `frontend/src/styles.css` -> Global Tailwind/PrimeNG configurations.
